# Biddy 시스템 아키텍처

## 전체 아키텍처

```
                        ┌─────────────┐
                        │   Client    │
                        │ (React 등)  │
                        └──────┬──────┘
                               │
                        ┌──────▼──────┐
                        │ API Gateway │
                        │   (:8000)   │
                        └──────┬──────┘
                               │
                 ┌─────────────┼─────────────┐
                 │             │             │
          ┌──────▼──┐   ┌─────▼────┐  ┌─────▼────┐
          │ Member  │   │ Product  │  │ Auction  │  ...
          │ (:8081) │   │ (:8082)  │  │ (:8084)  │
          └────┬────┘   └────┬─────┘  └────┬─────┘
               │             │             │
               └─────────────┼─────────────┘
                             │
                    ┌────────▼────────┐
                    │     Kafka       │
                    │    (:9092)      │
                    └─────────────────┘

  ┌──────────────┐                    ┌──────────────┐
  │  Discovery   │ ◄── 서비스 등록 ──► │   Config     │
  │   (:8761)    │                    │   (:8888)    │
  └──────────────┘                    └──────────────┘
```

## 서비스 구성

| 구분 | 서비스 | 역할 |
|---|---|---|
| 인프라 | Discovery (Eureka) | 서비스 등록/발견 |
| 인프라 | Config Server | 중앙 설정 관리 |
| 인프라 | API Gateway | 라우팅, 인증, CORS |
| 도메인 | Member | 회원, 인증, 예치금 |
| 도메인 | Product | 상품 등록/조회/검색 |
| 도메인 | Order | 장바구니, 주문 |
| 도메인 | Auction | 경매, 입찰, 낙찰 |
| 도메인 | Payment | 결제, 환불, 정산 |

## 기술 스택

| 레이어 | 기술 |
|---|---|
| Backend | Java 21, Spring Boot 3.4, Spring Cloud 2024 |
| Database | PostgreSQL 16 (서비스별 DB 분리) |
| Messaging | Apache Kafka 3.7 (KRaft 모드) |
| Container | Docker, Docker Compose |
| API 문서 | SpringDoc OpenAPI (Swagger) |

## 서비스 간 통신

| 통신 방식 | 용도 | 예시 |
|---|---|---|
| 동기 (REST) | 즉시 응답이 필요한 요청 | 상품 조회, 회원 인증 |
| 비동기 (Kafka) | 이벤트 기반 처리 | 낙찰 → 주문 생성, 결제 완료 → 상태 변경 |
