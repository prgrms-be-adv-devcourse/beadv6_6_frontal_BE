## 1. API 구현 현황

| # | API | Method | Path | 상태 | 테스트 |

> Auction Service에 POST API 불필요. Product가 데이터의 주인(Single Source of Truth).

## 2. 내부 기능 구현 현황

| 기능 | 상태 | 설명 |
|------|:----:|------|
| 비관적 락 (Pessimistic Lock) | O | `findByIdForUpdate()` — 입찰 동시성 제어 |
| 입찰 2단계 검증 | O | Pre-Lock → Post-Lock 검증 분리 |
| WebSocket STOMP 설정 | O | `WebSocketConfig` — STOMP + SockJS |
| WebSocket 입찰 push | O | `AuctionWebSocketPublisher.publishBid()` — BidService에서 호출 |
| WebSocket 종료 push | O | `AuctionWebSocketPublisher.publishEnded/Unsold()` — 스케줄러에서 호출 |
| 경매 종료 스케줄러 | O | `AuctionCloseScheduler` — 1초 간격, @Transactional으로 dirty checking 보장 |
| Kafka Consumer (Product) | O | `ProductAuctionRegisteredConsumer` — `product.auction.registered` 수신 → 경매 자동 생성 |
| 관심 경매 토글 | O | `WatchService` — 등록/해제 토글 + watcherCount 갱신 |
| 프론트 테스트 화면 | O | `static/index.html` — 바닐라JS, WebSocket STOMP 연동 |
| Kafka Producer (Auction → Order) | O | `AuctionEndedEventProducer` — `auction.ended` 발행 (낙찰 시 Order Service에 주문 생성 요청) |

## 5. DB 현황

### 5.1 auction 테이블

```sql
CREATE TABLE auction (
    auction_id      VARCHAR(20)   NOT NULL PRIMARY KEY,
    seller_id       BIGINT        NOT NULL,
    name            VARCHAR(200)  NOT NULL,
    brand           VARCHAR(100),
    edition         VARCHAR(100),
    category        VARCHAR(50),
    description     TEXT,
    thumbnail_url   VARCHAR(500),
    start_price     BIGINT        NOT NULL,
    min_increment   BIGINT        NOT NULL,
    current_bid     BIGINT        NOT NULL DEFAULT 0,
    bid_count       INTEGER       NOT NULL DEFAULT 0,
    watcher_count   INTEGER       NOT NULL DEFAULT 0,
    status          VARCHAR(10)   NOT NULL DEFAULT 'LIVE'
                    CHECK (status IN ('LIVE','ENDED')),
    ends_at         TIMESTAMP(6)  NOT NULL,
    created_at      TIMESTAMP(6)  NOT NULL,
    updated_at      TIMESTAMP(6)  NOT NULL
);
```

### 5.2 bid 테이블

```sql
CREATE TABLE bid (
    bid_id      BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    auction_id  VARCHAR(20)   NOT NULL,
    bidder_id   BIGINT        NOT NULL,
    amount      BIGINT        NOT NULL,
    bid_at      TIMESTAMP(6)  NOT NULL
);

CREATE INDEX idx_bid_auction_bid_at  ON bid (auction_id, bid_at DESC);
CREATE INDEX idx_bid_auction_amount  ON bid (auction_id, amount DESC);
```

### 5.3 auction_watch 테이블

```sql
CREATE TABLE auction_watch (
    watch_id    BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    auction_id  VARCHAR(20)   NOT NULL,
    member_id   BIGINT        NOT NULL,
    created_at  TIMESTAMP(6)  NOT NULL,
    CONSTRAINT uk_watch_auction_member UNIQUE (auction_id, member_id)
);
```

## 10. 경매 등록 방식 변경 이력

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 등록 주체 | Auction Service (POST /auctions) | **Product Service** (POST /products, type=AUCTION) |
| 통신 방식 | Auction → Product REST 호출 | **Product → Kafka (Outbox) → Auction Consumer** |
| 토픽 명 | `product.listed` | **`product.auction.registered`** |
| 데이터 주인 | 모호 | **Product Service** (brand, edition, category 관리) |
| 원자성 | 서버 다운 시 불일치 | **Outbox 보장** (자동 복구) |
| 구현 파일 | - | `ProductAuctionRegisteredConsumer.java`, `ProductAuctionRegisteredPayload.java` |

## 11. Kafka 현황 (확정)

| 방향 | Topic | 역할 | 상태 |
|------|-------|------|:----:|
| Product → **Auction** | `product.auction.registered` | 경매 자동 생성 (Consumer) | O |
| Auction → **Order** | `auction.ended` | 낙찰 주문 생성 (Producer) | O |

> **Auction 서비스의 Kafka 구현 완료.**
> 입찰 알림은 WebSocket 직접 push (Kafka 불필요).
> 결제 관련(`payment.completed`, `payment.timeout`)은 Order/Payment 서비스 간 흐름이므로 Auction 범위 아님.

### 12.9 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `psql: not recognized` | PATH 미설정 | `$env:PATH += ";C:\Program Files\PostgreSQL\18\bin"` |
| `public 스키마 접근 권한 없음` | biddy 사용자 권한 부족 | postgres로 접속 후 `GRANT ALL ON SCHEMA public TO biddy;` |
| `UnknownHostException: kafka` | Kafka `ADVERTISED_LISTENERS` 문제 | docker-compose에서 `PLAINTEXT://localhost:9092` 확인 |
| 스케줄러 무한 반복 | `@Transactional` 내부 호출 | `processExpiredAuctions()`에 `@Transactional` 선언 확인 |
| Docker `unable to get image` | Docker Desktop 미실행 | Docker Desktop 실행 후 재시도 |
| Eureka 연결 실패 경고 | Discovery 서비스 미실행 | 로컬 단독 실행 시 무시 가능 |
